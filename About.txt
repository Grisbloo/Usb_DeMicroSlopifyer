launcher.bat
powershell.exe -Command "Start-Process powershell.exe -Verb RunAs -ArgumentList '-ExecutionPolicy Bypass -File setup.ps1'"
the command to run: powershell.exe
it runs the command using -Verb RunAs to provide admin access
-ArgumentList tells exactly what to run (in this case: setup.ps1)

setup.ps1
$BloatwareList = @(
    "Microsoft.XboxApp",
    "Microsoft.XboxGamingOverlay",
    "Microsoft.XboxIdentityProvider",
    "Microsoft.XboxSpeechToTextOverlay",
    "Microsoft.ZuneVideo",
    "Microsoft.549981C3F5F10",
    "microsoft.windowscommunicationsapps"
)

is a list of all the easy to remove files that comes preinstalled on microsoft computers

foreach ($Target in $BloatwareList){
    Get-AppxPackage -Name $Target -AllUsers |
    Remove-AppxPackage -AllUsers -ErrorAction SilentlyContinue
}

a simple foreach loop that takes a name from the BloatwareList and for all users removes it
ErrorAction simply states that if there is an error, keep moving on (assuming the bloated element is not on the pc)

Stop-Process -name "OneDrive" -Force -ErrorAction SilentlyContinue

Stopping OneDrive, forcefully, and not allowing any errors

Start-Process -FilePath "$env:SystemRoot\SysWOW64\OneDriveSetup.exe" -ArgumentList "/uninstall" -Wait

finds the file of the OneDriveSetup and finds its uninstaller. which uninstalls and the powershell must wait for it to be over

Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders" -Name "Desktop" -Value "$env:USERPROFILE\Desktop"
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders" -Name "Personal" -Value "$env:USERPROFILE\Documents"

after removal of OneDrive, the registry needs to be updated with the new old folder locations. Personal is the new documents. 
This removes /OneDrive/Desktop and makes it just Desktop.
